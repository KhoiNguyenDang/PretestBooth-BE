# Authentication API Documentation

## Base URL

```
http://localhost:3000
```

## Overview

The Authentication API provides endpoints for user registration, login, token management, email verification, and password reset. The API uses JWT (JSON Web Token) for authentication on protected endpoints.

## Authorization Header

Protected endpoints require a Bearer token in the Authorization header:

```
Authorization: Bearer <access_token>
```

---

## Endpoints

### Register

```http
POST /auth/register
```

**Description:** Register a new user account. Only school email addresses are accepted.

**Request Body:**

```json
{
  "email": "12345678.yourname@student.iuh.edu.vn",
  "password": "password123"
}
```

**Validation Rules:**

- `email`: Must be a valid school email in the format `XXXXXXXX.yourname@(student|teacher).iuh.edu.vn`
- `password`: Minimum 6 characters

**Success Response (201):**

```json
{
  "message": "User registered successfully. Please check your email for verification."
}
```

**Error Response (400):**

```json
{
  "statusCode": 400,
  "message": ["Vui lòng sử dụng email trường (XXXXXXXX.yourname@(student|teacher).iuh.edu.vn)"],
  "error": "Bad Request"
}
```

---

### Login

```http
POST /auth/login
```

**Description:** Authenticate a user and receive access and refresh tokens.

**Request Body:**

```json
{
  "email": "user@student.iuh.edu.vn",
  "password": "password123"
}
```

**Validation Rules:**

- `email`: Valid email format
- `password`: Minimum 6 characters

**Success Response (200):**

```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": "uuid-here",
    "email": "user@student.iuh.edu.vn",
    "role": "STUDENT",
    "isEmailVerified": true
  }
}
```

**Error Response (401):**

```json
{
  "statusCode": 401,
  "message": "Invalid credentials",
  "error": "Unauthorized"
}
```

---

### Booth Activate

```http
POST /auth/booth-activate
```

**Description:** Activate a physical booth session using `boothCode` + one-time OTP generated by admin.

**Request Body:**

```json
{
  "boothCode": "BOOTH-01",
  "otp": "123456"
}
```

**Success Response (200):**

```json
{
  "boothId": "uuid-here",
  "boothCode": "BOOTH-01",
  "boothName": "Booth 01",
  "boothSessionToken": "eyJhbGciOi..."
}
```

**Error Cases:**

- OTP invalid/expired/used
- Booth not ACTIVE
- Exceeded max OTP attempts

**Note:** Booth session remains active until `POST /auth/booth-logout` is called.

---

### Booth Logout

```http
POST /auth/booth-logout
```

**Description:** Explicitly terminates booth session. After logout, existing booth token can no longer be used.

**Request Body:**

```json
{
  "boothSessionToken": "eyJhbGciOi..."
}
```

---

### Booth Session Status

```http
GET /auth/booth-session?boothSessionToken=eyJhbGciOi...
```

**Description:** Validate an active booth session token without side effects. Intended for kiosk boot-up and refresh checks.

**Success Response (200):**

```json
{
  "active": true,
  "booth": {
    "id": "uuid-here",
    "code": "BOOTH-01",
    "name": "Booth 01",
    "status": "ACTIVE"
  },
  "sessionActivatedAt": "2026-03-18T08:00:00.000Z",
  "sessionActivatedAtLocal": "2026-03-18 15:00:00"
}
```

**Error Cases:**

- booth session token invalid
- booth inactive or session revoked

---

### Generate Booth Activation OTP (Admin)

```http
POST /booths/activation-otp
Authorization: Bearer <admin_access_token>
```

**Description:** Admin creates a one-time OTP for a booth code. OTP is used by kiosk in `/auth/booth-activate`.

**Request Body:**

```json
{
  "boothCode": "BOOTH-01"
}
```

**Success Response (200):**

```json
{
  "boothId": "uuid-here",
  "boothCode": "BOOTH-01",
  "otp": "123456",
  "expiresAt": "2026-03-18T02:10:00.000Z"
}
```

### Booth Login

```http
POST /auth/booth-login
```

**Description:** Student login at an activated booth. Backend validates user credentials + booth session token + booking window and auto check-in.

**Request Body:**

```json
{
  "email": "user@student.iuh.edu.vn",
  "password": "password123",
  "boothSessionToken": "eyJhbGciOi..."
}
```

**Success Response (200):**

```json
{
  "accessToken": "eyJhbGciOi...",
  "refreshToken": "eyJhbGciOi...",
  "user": {
    "id": "uuid-here",
    "email": "user@student.iuh.edu.vn",
    "role": "STUDENT"
  },
  "booth": {
    "id": "uuid-here",
    "code": "BOOTH-01",
    "name": "Booth 01"
  },
  "checkedInBooking": {
    "id": "booking-uuid",
    "boothId": "uuid-here",
    "status": "CHECKED_IN",
    "startTime": "2026-03-18T08:30:00.000Z",
    "endTime": "2026-03-18T09:30:00.000Z"
  }
}
```

**Error Cases:**

- Booth session token invalid/expired
- User not STUDENT
- No valid booking by `user + booth + configured time window`


### Reset Password

```http
POST /auth/reset-password
```

**Description:** Reset password using the code received via email.

**Request Body:**

```json
{
  "email": "user@student.iuh.edu.vn",
  "code": "123456",
  "newPassword": "newpassword123"
}
```

**Validation Rules:**

- `email`: Valid email format
- `code`: Must be exactly 6 digits
- `newPassword`: Minimum 6 characters

**Success Response (200):**

```json
{
  "message": "Password reset successfully"
}
```

**Error Response (400):**

```json
{
  "statusCode": 400,
  "message": "Invalid or expired reset code",
  "error": "Bad Request"
}
```

## User Roles

The system supports three user roles:

- **STUDENT**: Regular student user
- **LECTURER**: Teacher/lecturer user
- **ADMIN**: Administrator with full access

---

## Notes

1. **Email Verification**: Users must verify their email before they can fully access the system. After registration, a verification email is sent automatically.

2. **Password Reset Flow**:
   - User requests password reset via `/auth/forgot-password`
   - System sends a 6-digit code to the user's email
   - User submits the code along with new password via `/auth/reset-password`
   - Reset code expires after a certain period

3. **Token Expiry**:
   - Access tokens have a short lifespan (typically 15-60 minutes)
   - Refresh tokens have a longer lifespan (typically 7-30 days)
   - Use the `/auth/refresh` endpoint to get a new access token

4. **School Email Requirement**: Registration requires a valid IUH school email address in the format:
   - Student: `XXXXXXXX.yourname@student.iuh.edu.vn`
   - Teacher: `XXXXXXXX.yourname@teacher.iuh.edu.vn`
